

# 用户提问完整流程分析

## 1. 请求接收阶段

函数调用链：

- app.py:chat() - 接收HTTP POST请求
- state\_machine.py:run\_chat() - 处理聊天逻辑
- langgraph:compiled\_graph.invoke() - 执行状态图
  详细流程：

1. 用户向 /chat 接口发送POST请求，包含 message 和 thread\_id
2. app.py:chat() 记录日志并调用 run\_chat()
3. run\_chat() 创建 HumanMessage 对象
4. 调用 compiled\_graph.invoke() 执行状态图，传入消息和线程ID

## 2. 主题检测与意图匹配阶段

节点： detect\_topic 函数： core/nodes.py:detect\_topic()

详细流程：

1. 获取最新用户消息
2. 调用大模型检测对话主题
3. 意图匹配 ：调用 intent\_manager.match\_intent() 进行意图识别
4. 返回 current\_topic 、 intent\_id 和 intent\_score
   意图匹配流程：

- intent\_manager.match\_intent() 计算用户输入的向量
- 在 intent\_index Milvus集合中进行相似度搜索
- 返回匹配度最高的 intent\_id 和相似度分数

## 3. 知识库检索阶段

节点： retrieve\_knowledge 函数： core/nodes.py:retrieve\_knowledge()

详细流程：

1. 获取最新用户消息和意图ID
2. 调用 knowledge\_base.search() 搜索知识库
3. 处理检索结果，提取最高相似度分数
4. 返回 knowledge\_results 、 highest\_score 和 intent\_id

## 4. 智能路由阶段

函数： core/edges.py:decide\_next\_node()

详细流程：

1. 获取知识库检索的最高相似度分数
2. 根据分数阈值决定路由方向：
      - 分数 > 0.4： direct\_answer （事实性回答）
      - 0.2 < 分数 ≤ 0.4： clarify\_question （操作性回答）
      - 分数 ≤ 0.2： fallback\_response （闲聊型回答）

## 5. 回答生成阶段

节点： direct\_answer / clarify\_question / fallback\_response 函数： core/nodes.py:direct\_answer() 等

详细流程：

1. direct\_answer ：基于知识库信息生成直接回答
2. clarify\_question ：生成澄清问题的回复
3. fallback\_response ：生成兜底回复

## 6. 上下文管理阶段

节点： increment\_rounds → check\_conversation\_rounds 函数： core/nodes.py:increment\_rounds() 和 core/edges.py:check\_conversation\_rounds()

详细流程：

1. 增加对话轮数
2. 检查对话轮数是否超过15轮
3. 如果超过15轮，进入 summarize 节点进行对话总结
4. 否则结束对话

## 7. 响应返回阶段

函数调用链：

- state\_machine.py:run\_chat() - 提取助手响应
- app.py:chat() - 构造并返回响应
  详细流程：

1. run\_chat() 从状态图结果中提取最新的助手响应
2. 构造包含 response 、 topic 和 thread\_id 的响应
3. app.py:chat() 返回 ChatResponse 对象

## 8. 流式响应流程（可选）

函数： app.py:chat\_stream()

详细流程：

1. 接收流式聊天请求
2. 创建 HumanMessage 对象
3. 调用 compiled\_graph.astream\_events() 获取流式事件
4. 过滤出大模型流式输出事件
5. 实时 yield 每个token块
6. 处理客户端断开连接的情况
