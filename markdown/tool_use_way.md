# 查询快递物流信息

**分类：** 杂项

买了东西想知道快递到哪儿了？这个接口帮你实时追踪物流状态。

## 功能概述
提供一个快递单号，系统会自动识别快递公司并返回完整的物流轨迹信息。这个接口目前可以查询中通、圆通、韵达、申通、极兔、顺丰、京东、EMS、德邦等主流快递公司的物流信息。

## 使用须知
- **自动识别**：不知道是哪家快递？系统会根据单号规则自动识别快递公司（推荐使用）
- **手动指定**：如果已知快递公司，可以传递 `carrier_code` 参数，查询速度会更快
- **手机尾号验证**：顺丰等部分快递公司需要验证收件人手机尾号才能查询详细物流，如果返回 `暂无物流信息`，建议尝试传入 `phone` 参数
- **查询时效**：物流信息实时查询，响应时间通常在1-2秒内

## API 端点

**方法：** GET
**路径：** /misc/tracking/query
**完整API地址：** https://uapis.cn/api/v1/misc/tracking/query
**文档页面：** https://uapis.cn/docs/api-reference/get-misc-tracking-query

## 响应

### 200 - 请求成功
查询成功！直接返回快递的完整物流轨迹。

**响应字段：**

- **`tracking_number`** (string) - 示例：`YT1234567890123`
  快递单号

- **`carrier_code`** (string) - 示例：`yuantong`
  快递公司编码

- **`carrier_name`** (string) - 示例：`圆通速递`
  快递公司名称

- **`track_count`** (integer) - 示例：`3`
  物流轨迹数量

- **`tracks`** (array[object])
  物流轨迹列表，按时间倒序排列

### 400 - 错误的请求
参数错误，请检查快递单号是否正确。

**响应字段：**

- **`code`** (string) - 示例：`INVALID_ARGUMENT`

- **`message`** (string) - 示例：`缺少快递单号参数`

### 404 - 未找到
当前没有查询到物流轨迹时会返回 404，并附带错误码和提示信息。如果返回此错误，建议尝试传入 `phone` 参数（收件人手机尾号）再次查询。

**响应字段：**

- **`code`** (string) - 示例：`NO_TRACKING_DATA`

- **`message`** (string) - 示例：`暂无物流信息`

## 查询参数

### tracking_number
- **类型：** string
- **描述：** 快递单号，通常是一串10-20位的数字或字母数字组合。
- **示例：** `YT1234567890123`
- **提示：** 输入完整的快递单号

### carrier_code
- **类型：** string
- **描述：** 快递公司编码（可选）。不填写时系统会自动识别，填写后可加快查询速度。
- **提示：** 例如：zhongtong、yuantong、yunda 等

### phone
- **类型：** string
- **描述：** 收件人手机尾号，4位数字（可选）。部分快递公司需要验证手机尾号才能查询详细物流信息。
- **约束：** 正则: ^\d{4}$
- **提示：** 收件人手机号后4位，如 2312

### full
- **类型：** boolean
- **描述：** 使用这个参数可以获得完整的物流信息。但会消耗28积分/一次（不过缓存命中半价）。因为成本实在太贵了，否则非常非常亏说是
- **提示：** 填写true即可打开此功能，不填或false则不生效
